# Security policy

Cogs is pre-release software. Do not use it to protect production credentials or untrusted workloads yet.

Report suspected vulnerabilities privately through [GitHub private vulnerability reporting](https://github.com/nenb/cogs/security/advisories/new). Do not include real credentials, private source, prompts, or session exports in an issue.

Nick Byrne (`@nenb`) is the initial security owner and reviewer. Security claims are valid only when linked to an applicability-aware report from an authoritative profile; the insecure container and optional macOS VM are never authoritative guest-root evidence.
